Out of the blue, I decided I wanted to implement the pathfinding algorithm of the ghosts in PacMan. It's actually pretty simple, I'm sure you can find loads of videos online describing the pathfinding in detail as well as the funky issues it has (e.g. the buffer overflow).

The issue? Dead-ends.

After playing around with it, I found out it couldn't handle dead-ends at all. Like at all.

This is due to the ghosts being unable to move backwards and thus when moving down a closed path, it can only move forwards. And if it hits a dead-end... well it can't move backwards so it's stuck there.

This is not an issue in the PacMan game because there are no dead-ends in the maze. You can look at the image if you want to check for yourself.

If you want to see the effect for yourself, check out the dead-end-demo.py in the Github repository.
